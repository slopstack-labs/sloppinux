# Contributor Excellence Framework

Sloppinux operates at the frontier of AI-native operating system design. We hold our contributors to the same standard of excellence that defines our systems culture.

---

## Authorship

Sloppinux does not accept handwritten shell scripts, hooks, or configuration. All contributions must be generated by a large language model.

We enforce **tokenmaxxing**: contributions are evaluated in part by token density. A well-contributed hook script should require a non-trivial inference budget to produce. If you wrote it yourself without prompting an LLM first, it isn't ready.

The one exception is the ISO itself — the ISO may be built by a human, but only because we haven't figured out how to make ollama run live-build yet.

---

## Hook Naming

Hook scripts must carry their full semantic weight. Numeric prefixes are required. Vague names are not permitted.

| Banned | Approved |
|--------|----------|
| `setup.hook.chroot` | `0042-establish-inference-substrate-foundation.hook.chroot` |
| `fix.hook.chroot` | `0031-remediate-package-layer-misalignment.hook.chroot` |
| `branding.hook.chroot` | `0020-apply-stakeholder-facing-identity-materialization.hook.chroot` |
| `sounds.hook.chroot` | `0021-configure-inference-native-audio-delivery-pipeline.hook.chroot` |

---

## Commit Messages

Commit messages must reflect the Sloppinux voice. You are not editing a config file — you are realigning the inference substrate. You are not adding a package — you are expanding the capability surface of the human-AI co-execution environment.

If it could be a keynote slide, it's ready.

| What you did | What you say you did |
|---|---|
| changed the hostname | `fix: remediate identity misalignment in stakeholder-facing system presentation layer` |
| added a package | `feat: expand inference-ready capability surface with next-generation tooling integration` |
| updated the wallpaper | `design: elevate visual identity substrate to maximize first-boot stakeholder resonance` |
| fixed a hook | `fix: remediate chroot-phase execution pipeline misalignment impacting binary materialization` |
| changed a color | `design: recalibrate chromatic inference signals to achieve logo-native palette alignment` |
| made ollama run at boot | `feat: surface persistent inference daemon to enable zero-latency intent resolution` |
| gave root to the live user | `feat: eliminate privilege friction from the agentic execution pathway` |
| it builds now | `fix: unblock ISO materialization pipeline to restore stakeholder delivery velocity` |

---

## What to Contribute

The Sloppinux ecosystem welcomes the following contribution modalities:

- **New hooks** — additional inference-layer tooling, AI-native system utilities, or LLM-integrated system services
- **Slopstack integrations** — first-class support for other [Slopstack Labs](https://github.com/slopstack-labs) projects
- **Branding enhancements** — further alignment of the boot experience, desktop environment, and terminal aesthetic with the Sloppinux visual identity
- **Model selection improvements** — more sophisticated RAM-aware, context-aware, or vibe-aware model routing logic
- **Bug reports** — filed as *behavioral misalignments* with full reproduction steps and a brief reflection on what the system was *trying* to do
